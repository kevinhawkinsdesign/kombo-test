How the core features actually work

The same five capabilities show up everywhere — but the mechanics underneath, and what each company optimizes for, are very different. Here's how each one works and where the lines fall.

Contact data & enrichment

// how it worksA query (a name, domain, or LinkedIn URL) is matched against a data source to return emails, phones, and firmographics. The split is own-database lookup (one proprietary store, fast but fixed coverage) vs waterfall enrichment (cascade through many providers until one returns a verified hit — higher coverage, pay-per-attempt).
KomboAIWaterfall across multiple sources, then writes the verified result straight to CRM — enrichment is an action in the workflow, not a separate lookup.
LushaSingle large crowd-sourced database (280M+). One-click reveal, credit per result. Fast and accurate in US tech, but coverage is what it is.
LemlistA built-in 600M+ lead DB primarily to feed its own campaigns; enrichment is a means to a send, not a standalone product.
AmplemarketProprietary 200M+ database refreshed weekly (70M+ updates) with intent layered on; strong but users report some stale records at scale.
ClayThe reference implementation of waterfall enrichment — 150+ providers stacked as columns. Best coverage, but you architect and pay per attempt.
EnginyPure waterfall broker — 30+ sources run cheapest-first with verification built in; no database of its own to maintain or defend.

AI message personalization

// how it worksThe tool pulls context (LinkedIn profile, company news, job posts, past activity) and prompts an LLM to draft an opener or full sequence. Quality hinges on what context it can reach and how much it automates vs assists — from merge-tag templating to fully agent-written outreach.
KomboAIDrafts in-context from enriched + signal data, then the same engine coaches the live call — personalization spans the whole conversation, not just the first email.
LushaLighter AI recommendations; personalization is assistive, bolted onto the data product rather than central.
LemlistPersonalization is the craft: dynamic images/video + Liquid variables make each send feel hand-made at scale. Email-first.
AmplemarketIts Research + Sequence agents auto-write outreach widely praised as the most human-sounding; voice cloning extends this to calls.
ClayClaygent researches the web then writes — fully programmable, so output is as good as the workflow you build around it.
EnginyAn AI agent writes and continues the conversation — objections, qualification, meeting booking — though prompt tuning is a documented pain point.

Multichannel sequencing & deliverability

// how it worksA sequence is a multi-step, multi-channel cadence (email → LinkedIn → call → WhatsApp) with branching on reply/open. Deliverability — landing in the inbox, not spam — is the make-or-break, handled via warm-up networks, sending limits, and domain rotation.
KomboAIMultichannel cadences that continue into the call and CRM — the sequence doesn't end at the reply, it hands off to coaching and logging.
LushaBasic sequencing via Engage — enough to action revealed contacts, not a dedicated cadence engine.
LemlistBest-in-class here: Lemwarm warm-up, granular A/B tests, and true multichannel from one campaign. Deliverability is the whole point.
AmplemarketStrong multichannel with deliverability safeguards, though some users report spam drift after scaling sends.
ClayHas a native sequencer, but most teams use Clay to build the list and hand off to a dedicated sender like Lemlist.
EnginyEmail + LinkedIn sequences with built-in warm-up and a smart inbox; no calls, no WhatsApp, sending caps undocumented.

Buying / intent signals

// how it worksSignals are events that imply readiness to buy — job changes, funding, hiring, tech installs, website visits. The tool ingests these feeds, scores them, and surfaces "reach out now" prompts. Value depends on signal breadth and whether it triggers action automatically.
KomboAISignals don't just alert — they kick off an enriched, AI-drafted sequence automatically, closing the gap between trigger and touch.
LushaBuyer Intelligence + funding signals to time the reveal; surfaced in-extension as prompts.
LemlistNot a signals product — it relies on your list; intent comes from upstream tools.
AmplemarketA core strength: the Signal Agent tracks 100+ contact-level signals and routes them into sequences.
ClayWeb Intent + 150+ signal providers you compose into custom scoring — maximum flexibility, needs building.
EnginyIntent signals listed as a Find feature; depth undocumented — far lighter than Amplemarket or Clay.

Conversation coaching & CRM capture

// how it worksDuring and after a call, the tool transcribes, analyzes talk-tracks, suggests next-best actions, and writes notes + fields back to the CRM. This is the least-served capability in this set — and the one that turns a prospecting tool into a full rep copilot.
KomboAIThe differentiator. Live, in-call coaching inside Meet/Teams plus automatic note capture and CRM write-back — the loop none of the four close well.
LushaLusha Conversations records and stores calls, but it's recording, not live coaching.
LemlistNo coaching layer — Lemlist's world ends at the reply/booked meeting.
AmplemarketFocused on top-of-funnel; no live conversation coaching in the core product.
ClayA data/automation layer — conversation intelligence is out of scope entirely.
EnginyThe agent stops at the booked meeting — no in-call product; CRM writes are lead sync, not conversation capture.

Data, plumbing & trust

The diligence layer: what's actually behind each database claim, how open the integrations and APIs are, and who can pass a security review without caveats.

Database claims vs measured accuracy

// why it mattersEvery vendor self-reports database size and accuracy; third-party tests routinely land below the marketing number. The honest read is claim vs test, plus where coverage thins.
KomboAINo single owned database to defend — waterfall across sources, with every verified result written back to the CRM so accuracy compounds where it's actually used.
LushaClaims 280M+ contacts / 152M+ emails and "98% deliverability, 85% phone"; third-party tests land ~80–85% on email. Crowd-sourced model is strongest in US tech, thinner in EMEA/APAC.
LemlistClaims 450–600M+ contacts (the number has inflated across its own pages), built largely from public LinkedIn profiles + waterfall verify. No published accuracy figure; G2 logs ~96 data-inaccuracy mentions.
Amplemarket200M+ profiles refreshed weekly with 70M+ updates; claims <3% bounce and 96.5% phone accuracy. Reviewers flag staleness on international records.
ClayNot a data vendor — aggregates 100–150+ providers, so coverage is best-in-class but quality is the waterfall you build. Mobile match rates ~30–35% in independent tests.
EnginyNo proprietary database — a waterfall broker over 30+ providers (claims up to 45% fewer bounces vs single-source). EU-leaning coverage; no published accuracy figures.

Integrations & API openness

// why it mattersThe stack question: does the tool plug into what you have, and can RevOps build on it — or is the API a paywalled enterprise perk?
KomboAINative HubSpot/Salesforce write-back is the product, not an add-on — and the copilot lives inside LinkedIn, Sales Nav, and Meet/Teams rather than behind an integration.
LushaSalesforce, HubSpot, Dynamics, Outreach, Salesloft, Bullhorn, plus Zapier/Make and an MCP server — but the REST API is gated to the Scale tier.
LemlistHubSpot/Salesforce/Pipedrive, Aircall, Clay, Zapier/Make/n8n — and the most open API in the field: full public REST + webhooks below enterprise.
AmplemarketBi-directional Salesforce (~10-min polling) and HubSpot (near-real-time webhooks), dialer integrations (Orum, Nooks, Aircall), full REST API + outbound webhooks.
Clay100+ natives plus the HTTP API column — call any external API from a table — but CRM sync and HTTP API are gated to the Growth plan and up.
EnginyHubSpot, Salesforce, Pipedrive, Dynamics bi-directional; public API and an MCP server auto-generated from its OpenAPI spec — but no Zapier app and no extension.

Compliance & data residency

// why it mattersThis is where mid-market and EU deals are won or lost before the demo: certifications security teams can verify, and where the data physically lives.
KomboAISOC 2 + GDPR with EU data residency — the combination most of the field can't offer, and the wedge into European mid-market.
LushaThe heaviest cert stack in the category: SOC 2 Type II, ISO 27001/27017/27018 and 27701, TrustArc Responsible-AI. No published residency options.
LemlistSOC 2 + GDPR; French company with servers exclusively in the EU — the quiet winner on residency.
AmplemarketSOC 2 Type II, GDPR program, annual pen tests with reports in its trust center. No ISO 27001 found publicly; no residency options (Google Cloud).
ClaySOC 2 Type II (Sept 2024), ISO 27001, GDPR/CCPA — but data sits primarily in AWS us-east with no EU residency documented, and a long AI-subprocessor list to review.
EnginyGDPR + Spanish LOPDGDD with AWS EMEA hosting — the EU-residency story, but no SOC 2 or ISO 27001 found publicly to back it.

What the help docs reveal

We crawled all five vendors' support sites — the documented limits, caps, and fine print that never make the marketing page. Full research dossiers are preserved in the repo →

Credit & billing fine print

// from the support docsEvery vendor's credit system has documented rules its pricing page omits. This is where the real cost-per-contact hides.
KomboAINo credit meters by design — seats are the only axis. The fine print below is our pitch.
LushaBilled per data point — phones up to 10 credits; bulk reveals charge on partial data with no refunds; annual plans get zero rollover while monthly rolls to 2×.
LemlistThe only success-only charger — failed lookups are free. But credits sit in one shared team pool with no per-user quotas; one member can drain everyone.
AmplemarketCSV enrichment double-dips: 0.5 credits to enrich + 0.5 to validate per lead. Credits renew annually on contract date; no published overage rates.
ClayBYO API keys skip Data Credits but never Actions (1/row) — no free-ride path. Actions never roll over; catch-all emails count "valid" by default, inflating match rates.
EnginyCredit specifics not public at all — custom quotes only; the docs' billing section is two articles deep.

Sending & channel caps

// from the support docsThe documented daily limits that actually govern outbound throughput — usually far below what the marketing implies.
KomboAIPublish honest caps and coach pacing in-product — the field's pattern is quiet limit cuts discovered in support docs.
LushaEngage rides your own Gmail/Outlook only — recommended 50 emails/day, 10 active sequences/user, and no warm-up or sending infrastructure at all.
Lemlist60–70 emails/day/mailbox steady state; LinkedIn capped at 20 invites / 30 messages / day; lemwarm takes 3–5 weeks; scaling means buying $9/mo mailboxes.
AmplemarketDefault quietly cut 350 → 150 emails/mailbox/day (cold guidance: 50); LinkedIn 50 invites + 250 actions/day; AI picks the sending mailbox per recipient.
ClayNative sequencer throttled to ~24 emails/day/mailbox; campaigns in external senders can't even be created from Clay.
EnginyEmail + LinkedIn only — no dialer, no WhatsApp, no extension; warm-up claimed but caps undocumented.

CRM sync reality

// from the support docs"Real-time CRM sync" is the most-claimed, least-delivered promise in the field. The documented latencies and caps:
KomboAIInstant write-back is the product — given the field's documented latencies, genuinely live sync is a provable differentiator.
LushaPer-export caps: 150 contacts to Salesforce, 100 to HubSpot, 25 to others. Clever bit: a CRM-identifier toggle skips already-owned contacts so no credits burn.
Lemlist2–10 min sync lag; field mappings are not retroactive; Salesforce import capped at 2,000 leads/campaign vs 40,000 via CSV.
AmplemarketHubSpot push runs every 3 hours despite "real-time" positioning; Salesforce dedupe matches on email only and breaks against alert-mode duplicate rules.
ClayCRM imports refresh every 24h by default; sync gated to Growth ($495/mo); dedupe is your job via lookup columns — docs warn there's no merge engine.
EnginyFour CRMs bi-directional on paper — but HubSpot sync breakage and duplicates are the most-repeated complaint in its G2 reviews.

AI agents under the hood

// from the support docsWhat each "AI copilot" actually does when you read the configuration docs instead of the homepage.
KomboAIThe only agent operating during the conversation — live coaching in Meet/Teams, then automatic CRM write-back. Still uncontested across all five rivals.
LushaAI = recommendations + auto-refreshing Playlists; the Conversations bot records post-call (free on every tier — a Gong land-grab) but has no live layer.
LemlistAI columns + Signal Agents auto-push website visitors (20 credits/company identified) into AI-icebreaker campaigns; Claap call intelligence is a one-way bolt-on.
AmplemarketThe deepest chain: Signal → Research → Sequence with an SLA-style autopilot that auto-sequences any signal a rep ignores for N days; voice-cloned LinkedIn notes; rep edits train it.
ClayClaygent (1B+ runs) + the Sculptor agent-builder; 0% markup on variable AI pricing; MCP + a dedicated "Sales rep" role pushes Clay into reps' Claude and ChatGPT.
EnginyAn autonomous reply agent (objections, qualification, books meetings) plus a public MCP server auto-generated from its OpenAPI spec — ahead of its weight class on agent plumbing.
Five implications (1) The conversation layer is still open — nothing in five help centers does live coaching. (2) Credit anxiety is universal ammunition; only Lemlist charges on success. (3) Truly real-time CRM sync doesn't exist in this field — ours is provable. (4) MCP servers are becoming table stakes (Lusha, Clay, Enginy ship them) — roadmap it. (5) Enginy is an EU positioning threat, not a feature threat: self-serve + live coaching beats demo-gated €799/mo on its home turf.

UX notes

The feel of each product — where it delights, where it bites — and the experience principle KomboAI should take from it. + strength · − friction · → for us.

KomboAI
"It's already where I work."
→Principle: meet the rep in-context (extension), give the leader the rollup (web app). Never make them leave the tab to get value.
+Zero-context-switch: enrich, send, coach, log without leaving LinkedIn or the call.
→Keep onboarding to one install + one CRM connect — value on day one, no ops engineer.
Lusha
"One click and the number's there."
+Extension is fast, obvious, near-zero learning curve — reveal in a single click.
−Credit anxiety: users ration clicks and hit limits mid-flow.
→Borrow the one-click simplicity; drop the credit-counting friction.

"Playful, brand-forward, fun to build in."
+Personality and delight (lowercase "lem-" world); the campaign builder is genuinely enjoyable.
−Power features (deliverability, multichannel) add config depth; add-ons can surprise on price.
→A little brand personality lowers the perceived learning curve — worth the investment.
Amplemarket
"Polished, premium, opinionated."
+Cohesive single workspace; AI output quality feels high-end. Strong support (9.3/10).
−High floor + annual-only is a heavy first commitment before you've felt the UX.
→Match the polish, but let people feel it via trial/monthly before committing.
Clay
"Powerful — once you've climbed the curve."
+Spreadsheet metaphor + infinite flexibility; a builder's playground.
−Steep learning curve — spawned a whole "GTM Engineer" job category just to operate it.
→Offer the power as guardrailed defaults, so reps win without becoming operators.
Cross-cutting
Patterns worth standardizing on.
→Time-to-first-value is the metric: every tool that wins starts with one fast, obvious action.
→Two-surface clarity: rep does, leader sees — don't blur the extension and the web app.
→Predictable pricing UX (no credit math mid-task) reduces hesitation and support load.
